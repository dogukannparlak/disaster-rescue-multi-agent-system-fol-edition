Evaluation Metrics

Performance analysis of the multi-agent rescue operation

Performance Scores

Key indicators measuring the quality of task assignments and execution

Capability Match Rate
—

Percentage of tasks assigned to a robot whose capability matches the task requirement. A score of 100% means every task was given to a robot that can perform it.

Priority Order Score
—

Measures whether tasks are executed in the correct priority order (high → medium → low). Calculated as the ratio of correctly ordered consecutive task pairs.

Task Distribution Balance
—

Measures how evenly tasks are distributed across robots. Based on Coefficient of Variation (CV). Score close to 100% = even distribution; lower scores indicate imbalance.

Completion Rate
—

Percentage of tasks that were successfully completed by the assigned robots. 100% means all tasks were executed.

Unassigned Tasks

—

Number of tasks that could not be assigned to any robot. Ideally this should be 0. Unassigned tasks indicate missing robot capabilities or system errors.

Cost Ratio (Actual / Optimal) — HW2

PDF: Cost(r,t) = Distance(r,t) / Speed(r); Efficiency = Actual Cost / Optimal Cost

Actual Total Cost
—

Sum of Distance(r,t) / Speed(r) for all task–robot pairs based on the actual assignment. Robot start position: (0, 0).

Optimal Total Cost
—

Minimum possible cost if each task is assigned to the lowest-cost compatible robot. With one robot per capability type, this equals the actual cost.

Cost ratio (1.0 = optimal)
—

Actual Cost / Optimal Cost (CSE419 Term Project #2). Value 1.0 means actual equals the per-task greedy lower bound; values > 1.0 indicate higher-than-baseline cost.

Metric Formulas

📊 Capability Match Rate

matched_tasks / total_tasks

📊 Priority Order Score

correct_pairs / total_pairs

📊 Task Distribution Balance

1 / (1 + CV) where CV = stdev / mean

📊 Completion Rate

completed_tasks / total_tasks

📊 Cost ratio (HW2)

Actual Cost / Optimal Cost (PDF)
where Cost(r,t) = Distance(r,t) / Speed(r)
