Agent Descriptions

📋 Planner Agent

Analyzes the disaster environment and produces a prioritized rescue plan. Orders tasks by urgency (high → medium → low) and provides justification for each decision.

Input: Task list, Robot fleet, Grid dimensions
Output: Plain-text rescue plan narrative

🔗 Assignment Agent

Reads the rescue plan and assigns each task to exactly one robot based on capability matching. Ensures high-priority tasks are queued first for each robot.

Input: Rescue plan, Task list, Robot fleet
Output: JSON assignment map {task_id → robot_id}

🔍 Critic Agent

Validates the assignment map for errors: capability mismatches, priority violations, and unassigned tasks. Automatically corrects any issues found.

Input: Assignment map, Task list, Robot fleet
Output: Critique text + corrected assignments

🤖 Robot Agents

Execute assigned tasks in priority order. Each robot processes its task queue sequentially, with execution time based on robot speed (fast: 1s, medium: 2s, slow: 3s).

Input: Assigned task queue
Output: Completed tasks + execution log

Implementation Mode

HW2 — FOL Mode

Symbolic Reasoning

Agents apply First-Order Logic axioms directly. No external server — fully deterministic and completes in under 1 second. Introduced in Term Project #2.

  • ∀t (Task(t)→∃!r Assigned(t,r)) — unique assignment
  • ∀t,r (Assigned(t,r)→Compatible(…)) — capability match
  • Priority(t1)>Priority(t2) → ScheduledBefore(t1,t2) — priority ordering
  • Cost(r,t) = Distance(r,t) / Speed(r) — cost optimisation
